精通SpringBoot

SpringBoot版本： 1.5.19.RELEASE

一、Spring Boot概述

传统开发方式回顾

开发过程

  1. 新建maven工程
  2. 根据项目可能需要的功能集成pom依赖
    • 需要丰富的开发经验
    • 自行选择jar包版本
    • 自行解决jar包版本冲突问题
  3. 创建spring配置文件（通常为了易于维护，会按照职责范围定义不同的ApplicationContext文件）
    • 配置事务切面
    • 配置日志切面
    • 配置数据库层
    • 配置缓存层（可选）
    • 配置文件服务层
    • 配置dubbo层（可选）
    • 配置sms文件
    • …（反正就是配置、配置）
  4. 创建spring mvc配置文件
    • 定义Controller层
    • 定义异常处理层
    • 常用的handler拦截器的初始化
  5. 配置web.xml文件
    • 配置web工程启动监听器类
    • 配置常用的filter
    • 配置spring mvc的入口servlet
  6. 开发业务功能
  7. 参数调优，生成war包，发布至外部web容器

面临的问题

Spring Boot开发方式

以下都是基于Maven方式构建，版本使用的是eclipse自带的3.3.9

what

  • Spring Boot相较于传统方式，可以快速的构建生产就绪的基于spring的应用程序。
  • Spring Boot内嵌了tomcat容器，用户只需要jar -jar运行应用即可。

why

  • 快速构建spring应用程序
  • 开箱即用
  • 提供丰富特性：内嵌服务器、metrics、健康检查、外部化配置等
  • 无需XML配置

how

  • 引入spring-boot-starter-parent

    #

    org.springframework.boot spring-boot-starter-parent 1.5.19.RELEASE

  • 配置maven构建插件（打包成可执行jar）

    #

    org.springframework.boot spring-boot-maven-plugin

  • 如果需要开发web工程，则引入spring-boot-starter-web

    #

    org.springframework.boot spring-boot-starter-web

  • 编写应用启动器

    #

    @SpringBootApplication

    public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
  • ssm工程搭建完毕

Spring Boot带来的挑战

  1. Spring Boot采用注解方式的配置Bean（传统习惯使用xml方式配置）
  2. 如果要熟练使用，那么对ssm工作机制都要熟练于心
  3. 开发中碰到问题不好排查

如何应对这些挑战

Spring基础回顾

  1. Spring注解开发

    • 注解三剑客
      • @Configuration
      • @Bean
      • @ComponentScan
    • 向Spring容器中添加组件
      • 包扫描 + 组件标注注解（@Controller/@Service/@Component/@Repository）
      • @Bean（导入第三方包中的类）
      • @Import
      • 使用spring提供的FactoryBean注册bean
    • bean生命周期管理
      • 创建 –> 初始化 –> 销毁
      • 几种管理方式
        • @Bean注解 指定初始化initMethod 和 销毁方法destoryMethod
        • 通过让bean实现InitializingBean（实现初始化逻辑） 及 DisposableBean（实现销毁逻辑）
        • 使用JSR250规范: @PostConstruct、@PreDestroy
        • 实现BeanPostProcessor [bean后置处理器]
    • 如何读取配置文件
      • 注解方式
        • @PropertySource加载配置文件，如果是多个使用@PropertySources注解
        • @Value
      • 编码方式
        • applicationContext.getEnvironment() –> ConfigurableEnvironment –> getProperty(“app.name”)
    • Bean的自动装配
      • @Autowired [Spring规范，推荐]
        • 优先根据类型去装配
        • 如果有多个此类型的实现类，那么按照 属性名 作为 bean name去匹配
        • 通过@Qualifier(“beanID”)去指定具体装配的bean实例
        • 默认情况下，@Autowired注解标注的属性，必须要装配，否则会报错. @Autowired(required=false)表示如果没有匹配的则不用去匹配
        • 作用域
          • 方法
          • 构造器（如果构造器只有一个参数，那么@Autowired注解可以省略，组件在初始化时，spring会自动注入构造器中的参数）
          • 属性
          • 方法参数
      • @Primary ==> 与@Bean注解配合使用，标注当前bean为自动装配时首选的bean，则缺省情况下会装配当前bean
      • @Qualifier ==> 指定装配的具体bean实例
      • JSR250规范
        • @Resource
        • @Inject
      • 实现xxxAware接口，完成xxx的注入
      • @Profile ==> 指定当前适用的环境
        • 激活方式
          • 运行时参数： spring.profiles.active=prod
          • applicationContext.getEnvironment().setActiveProfiles(“test”)
    • 面向切面编程（AOP）
      • 在程序运行期间动态的将某段代码切入到指定方法指定位置进行运行的编程方式
      • 实现机制 => 动态代理
      • 通知方法
        • 前置通知(@Before)
        • 后置通知(@After)
        • 返回通知(@AfterReturning)
        • 异常通知(@AfterThrowing)
        • 如果需要通过编码方式，控制前、后等 ==> 环绕通知(@Around)
      • 实现步骤
        • 定义切面类（在Class上添加注解@Aspect）
        • 在切面类中定义切入点表达式@PointCut(“execution(public * com.binbin.*(..))”)
        • 在切面类中 的目标方法 标注 通知注解
        • 在配置类中开启使用aop注解 => @EnableAspectJAutoProxy

  9. 过滤器/拦截器/AOP

    • 过滤器

      • Filter是依赖于Servlet容器，属于Servlet规范的一部分
      • Filter的执行由Servlet容器回调完成
      • Filter的生命周期由Servlet容器管理
      • 开发方式
        • 传统SSM架构
          • 在web.xml中，配置过滤器
        • Spring Boot方式
          • 使用注解 @WebFilter 与 @ServletComponentScan 配合使用【@WebFilter是Servlet3.0规范】
          • 通过FilterRegistrationBean完成注册（@Configuration注解类中）【spring boot没有web.xml，所以需要通过注册类将过滤器注册到容器中】

    • 拦截器

      • 拦截器独立于web容器，可以在任何情况下使用

      • 拦截器通常通过动态代理的方式来执行

      • 开发方式

        • 继承HandlerInterceptorAdapter 或者 实现HandlerInterceptor接口

        • 实现拦截业务逻辑

          • preHandle
          • postHandle
          • afterCompletion

        • 注册拦截器

          • 传统SSM架构（基于xml）

            #

            <mvc:interceptors>
    <mvc:interceptor>
        <mvc:mapping path="/**"/>
        <bean class="com.sdusz.framework.web.session.interceptor.RequestInterceptor">
            <property name="userSessionkey" value="userinfo"></property>
        </bean>
    </mvc:interceptor>
</mvc:interceptors>

          • Spring Boot方式

            • 继承WebMvcConfigurerAdapter，并覆写addInterceptors，来完成拦截器注册
              #### 

              @Configuration
@ComponentScan(basePackages = { "com.sdusz.framework.thirdlogin" })
public class InterceptorRegistryConfig extends WebMvcConfigurerAdapter {

    @Override
    public void addInterceptors(InterceptorRegistry registry) {

        registry.addInterceptor(new BootHandlerInterceptor());
    }

    @Bean
    public SpringContextHolder springContextHolder() {

        return new SpringContextHolder();
    }

}

    • AOP

      • 过滤器 / 拦截器 都是基于web请求的拦截
      • AOP可以实现基于类或者接口方法的拦截【与是否是web无关】
      • 开发方式： 参照上面spring基础部分

    • filter/interceptor/aop

      • 对比
        • filter是servlet规范，拦截的是web url请求，因此它不能获取到Spring相关调用信息，比如Controller，方法及参数等
        • interceptor是基于spring实现的，因此其可以获取到调用类、调用方法及调用参数的类型定义，但是无法获取参数值
        • aop可以拦截到方法的参数内容
      • 顺序
        • 过滤器 > 拦截器（HandlerInterceptor）> AOP切面

  10. Spring Boot异常处理

    • 基于@ControllerAdvice（可指定特定包） + @ExceptionHandler 组合
    • 实现HandlerExceptionResolver接口（全局）

  11. 如何编写起步依赖starter工程

    • 自动装配实现机制

      • @Configuration
      • @Conditional
        • ConditionalOnBean（spring已装配某个bean时，则当前bean也自动装配）
        • ConditionalOnMissingBean（spring未装配某个bean时，则当前bean也自动装配）
        • ConditionalOnClass（当前classpath下有这个类的依赖时，则当前bean也自动装配）
        • ConditionalOnMissingClass（当前classpath下无这个类的依赖时，则当前bean也自动装配）
        • ConditionalOnExpression（当SpEL表达式返回true时，自动装配当前bean）
        • ConditionalOnProperty（当配置文件中属性存在 或 等于某个值时，自动装配bean）
        • ConditionalOnResource（当某个资源存在时，则装配bean）
        • conditionConditionalOnWebApplication（是web环境时装配bean）
        • ConditionalOnNotWebApplication（非web环境时装配bean）
        • ConditionalOnJava（当前jvm的版本号满足条件时，装配bean）
      • 添加默认配置项：@ConfigurationProperties + @EnableConfigurationProperties

    • 命名规范约束

      • spring官方
        • spring-boot-xxx-starter
      • 自己开发的起步依赖
        • xxx-spring-boot-starter

    • 添加依赖

      #

      <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-autoconfigure</artifactId>
</dependency>

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-configuration-processor</artifactId>
    <optional>true</optional>
</dependency>

    • starter起步依赖工程中添加文件 META-INF/spring.factories

      • 配置org.springframework.boot.autoconfigure.EnableAutoConfiguration
